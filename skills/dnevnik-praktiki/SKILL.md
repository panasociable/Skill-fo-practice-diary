# Дневник практики (ПГТУ)

Скилл превращает свободный рассказ студента в готовый «Дневник практики»
по официальному шаблону ПГТУ. Студент может просто наговорить или напечатать
информацию о себе и практике — задача скилла вытащить из этого все нужные
поля, спросить про недостающие и собрать `.docx`.

## Как работает

1. Понять из сообщения студента, какие данные уже есть.
2. Спросить про недостающие обязательные поля — **одним коротким списком**,
   не растягивая на много сообщений.
3. Собрать JSON и запустить `scripts/fill_diary.py`.
4. Отдать готовый файл студенту через `present_files`.

Шапка документа (министерство, ПГТУ, Йошкар-Ола), блоки инструктажей,
раздел «Результаты прохождения практики» и формулировки 8 пунктов плана —
зашиты в шаблон и менять их не нужно.

## Какие данные собрать

Обязательные:

- `fio` — ФИО студента полностью (Фамилия Имя Отчество)
- `kafedra` — кафедра / факультет (например: «кафедра информационных систем»)
- `spec` — специальность/направление с кодом (например: «09.03.02 Информационные системы и технологии»)
- `kurs` — курс (число)
- `gruppa` — группа (например: «ИСТ-31»)
- `mesto` — место прохождения практики (организация)
- `start_date`, `end_date` — сроки практики, формат `ГГГГ-ММ-ДД`
- `zadanie` — индивидуальное задание / содержание отчёта (1-2 предложения)

Со значением по умолчанию (спрашивать, только если студент сам не сказал,
иначе подставить дефолт молча):

- `vid` — вид практики (по умолчанию «производственная»)
- `tip` — тип практики (например «технологическая»)
- `forma` — форма обучения (по умолчанию «очная»)

Опциональные (если студент не знает — оставить пустыми, в документе будут
прочерки для ручного заполнения):

- `instr_vuz` — должность и ФИО ответственного за инструктаж на кафедре
- `instr_org` — должность и ФИО ответственного в организации (например «директор – Иванов И.И.»)
- `prikaz_n`, `prikaz_date` — номер и дата приказа
- `dog_n`, `dog_date` — номер и дата договора

## Уточняющие вопросы

Не допрашивай студента по одному полю. Собери всё, что уже понятно из его
сообщения, и задай **один** вопрос со списком только недостающих обязательных
полей. Про опциональные (приказ, договор, ФИО ответственных) спрашивай мягко
и сразу предупреждай, что их можно пропустить и вписать от руки.

Даты сроков нужны в формате `ГГГГ-ММ-ДД`. Если студент говорит «с 29 июня по
12 июля», преобразуй сам, год бери текущий, если не указан иначе.

## План работ

План — стандартные 8 пунктов из шаблона. Их текст менять не нужно. Даты по
каждому пункту скрипт **сам** распределяет равномерно по сроку практики, так
что отдельно их спрашивать не надо.

## Сборка документа

Запиши собранные данные в JSON (структуру и пример смотри в
`assets/data_example.json`) и запусти:

```bash
python3 scripts/fill_diary.py <data.json> "<Дневник_Фамилия.docx>"
```

Скрипт использует `assets/template.docx`, требует `python-docx`
(`pip install python-docx --break-system-packages`). Имя выходного файла —
с фамилией студента, чтобы его было удобно найти.

После сборки **обязательно** покажи готовый `.docx` студенту через
`present_files`.

## Проверка

Перед выдачей убедись, что в документе не осталось текста вида `{{...}}` —
это означало бы незаполненное поле. Скрипт подставляет все плейсхолдеры; если
что-то осталось, значит в JSON не хватает ключа.
